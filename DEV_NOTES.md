# DEV_NOTES.md

## v0.1 - Начало проекта
**Что мы сделали:**
Мы разработали **AI Layout Assistant** — инструмент для автоматической генерации верстки (HTML/CSS) на основе текстового описания.

### Основные фичи:
- **Интеграция с DeepSeek (OpenRouter)**: Потоковая генерация ответов (streaming) для мгновенного отображения прогресса.
- **Интерактивное превью**: Результат рендерится в реальном времени внутри iframe через `srcdoc`.
- **Настройка "Золотого стандарта"**: Пользователь может задать эталонные HTML и CSS паттерны, которым ИИ будет следовать (методология БЭМ).
- **Управление сессией**: Кнопка "Остановить генерацию" через `AbortController`.
- **Сохранение состояния**: API-ключ и настройки паттернов сохраняются в `localStorage`.

### Какие решения принял и почему:
1. **Потоковая передача (Streaming)**: Выбрана для улучшения UX. Пользователь видит результат сразу.
2. **Маркеры разделения (`###CSS###`, `###HTML###`)**: Позволяют парсить код "на лету" без JSON-парсинга всего ответа.
3. **Iframe Preview**: Изолирует стили компонента от стилей приложения.

## v0.2 - Интеграция Word и SEO (Текущая версия)
**Что мы сделали:**
Значительно расширили возможности инструмента, превратив его в рабочий комбайн для контент-менеджера.

### Основные фичи v0.2:
- **Word (.docx) Integration**: Поддержка загрузки файлов Word через `mammoth.js`. Текст извлекается на клиенте.
- **Drag & Drop UI**: Интерактивная зона сброса файлов, заменяющая текстовое поле.
- **Smart Text Cleaning**: Авто-очистка текста от лишних пробелов и пустых строк (нормализация).
- **SEO Schema (JSON-LD)**: Генерация микроразметки Schema.org в отдельной вкладке.
- **Flicker-Free Preview**: Троттлинг (150ms) обновления iframe для устранения мерцания.
- **Universal Free Router**: Переход на `openrouter/free` для стабильности.

### Какие решения принял и почему:
1. **Троттлинг превью**: Рендеринг раз в 150мс убирает эффект "стробоскопа", сохраняя динамику.
2. **Удаление `<think>` тегов**: Фильтр для совместимости с Reasoning-моделями (R1).
3. **Структурные правила**: Обучили ИИ использовать "Золотой HTML" как единую обертку, а не плодить секции.
4. **Нейтральный UI**: Замена агрессивных анимаций на спокойное "дыхание" индикатора.

## v0.3 - Режим коррекции и правок (Refinement Mode)
**Что мы сделали:**
Добавлена возможность вести диалог с нейронкой для уточнения верстки без потери контекста.

### Основные фичи v0.3:
- **Refinement Chat**: Поле ввода правок под панелью результатов. Можно просить "сделай шрифт больше" или "поменяй цвет кнопок".
- **Context-Aware Refinement**: При правках ИИ получает текущий HTML, CSS и исходный текст, чтобы внести точечные изменения.
- **Smart UI Flow**: Панель правок появляется автоматически только после завершения основной генерации.
- **Enter-to-Send**: Быстрая отправка правок через Enter (с поддержкой Shift+Enter для новых строк).
- **SEO Schema Selector**: В настройках теперь можно выбрать тип микроразметки (Article, Product, FAQ, Recipe) или оставить "Авто", чтобы ИИ сам определил тип контента.
- **Structural Controls**: Добавлены переключатели для управления оберткой: можно включить/выключить использование тега `<section>` и наличие центрирующего контейнера `.container`.
- **Auto-Update on Config**: Если изменить настройки структуры или схемы при уже готовой верстке, ИИ автоматически перегенерирует код с учетом новых параметров сразу после сохранения настроек.
- **Resizable Panels**: Между левой и правой панелями добавлен интерактивный разделитель. Теперь можно вручную менять ширину рабочей области (текста) и области предпросмотра.
- **AI Response Validation**: Добавлены проверки на пустые ответы и некорректный формат (отсутствие маркеров). Теперь система выдает понятную ошибку вместо пустого результата.
- **UI Improvements**: Добавлен таймер генерации, детализированные статусы этапов и улучшенное окно обратной связи.

### Какие решения принял и почему:
1. **Сохранение контекста**: ИИ передается текущий код, чтобы он мог действовать как редактор, а не генерировать всё с нуля.
2. **Динамический UI**: Скрываем поле фидбека в режиме ожидания, чтобы не перегружать интерфейс.
3. **Бесшовный Loader**: Внедрена система индикации загрузки в кнопки для визуального отклика.
4. **Конфигурируемость SEO**: Дали пользователю контроль над типом схемы, так как "автоматика" не всегда угадывает намерение.
5. **Гибкость структуры**: Опции `section` и `container` позволяют адаптировать верстку под разные CMS и фреймворки (где-то обертки уже есть в шаблоне, где-то их надо генерировать).

## Что не работает и какие гипотезы
1. **Сложные таблицы из Word**: Mammoth теряет форматирование таблиц. *Гипотеза*: Использовать `convertToHtml` и стилизовать готовый HTML.
2. **Дублирование CSS**: ИИ иногда повторяет стили из эталона. *Гипотеза*: Нужен пост-процессинг или более строгий негативный промпт.
3. **Длинные диалоги**: Пока нет истории чата (только текущее состояние). *Гипотеза*: Добавить массив сообщений для полноценного чата.

## Что узнали в процессе
- **OpenRouter Free**: Модели часто меняют ID, поэтому `openrouter/free` — самый надежный вариант.
- **Refinement**: Передача текущего кода в `system` промпт работает стабильнее, чем просто просьба исправить.
